# Groq API Integration - Fix Summary

## Issue Fixed
The application was returning empty flashcards or mock responses instead of calling the Groq API.

## Root Causes
1. **Groq JSON Schema Format**: Groq doesn't support the same strict `json_schema` parameter as OpenAI
2. **Request Body Encoding**: Special characters (accents, Unicode) in JSON were causing 400 Bad Request errors when sent via curl

## Solutions Implemented

### 1. Updated LLM Client (`app/core/llm.py`)
- Changed from `response_format: {"type": "json_object", "schema": json_schema}` to `response_format: {"type": "json_object"}`
- Added schema instructions directly to the system prompt so Groq understands the expected JSON structure
- This ensures Groq generates properly formatted JSON without strict schema enforcement

### 2. Updated Dependencies (`requirements.txt`)
- Updated `groq` from `0.12.0` to `0.15.0` (fixes httpx compatibility issues)
- Updated `httpx` from `0.28.1` to `0.27.0` (compatible with Groq 0.15.0)

### 3. Request Encoding
- Use JSON files with UTF-8 encoding for requests with special characters
- Or use PowerShell's `ConvertTo-Json` for proper encoding

## Testing

### Working Request (with JSON file)
```powershell
$body = Get-Content request.json -Raw

Invoke-WebRequest -Uri "http://localhost:8000/generate_flashcards" `
  -Method POST `
  -Headers @{"Content-Type"="application/json"} `
  -Body $body `
  -UseBasicParsing | Select-Object -ExpandProperty Content
```

### Sample request.json
```json
{
  "topic": "Italiano",
  "text": "Il passato prossimo si usa per azioni concluse nel passato. Si forma con il presente del verbo ausiliare (avere o essere) e il participio passato del verbo principale.",
  "num_cards": 5
}
```
